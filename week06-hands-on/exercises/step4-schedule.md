# Step 4: Windows 작업 스케줄러 (20분)

> 지금까지 만든 `run_pipeline.py`를 **매일 오전 11:30에 Windows가 알아서 실행**하게 등록합니다.
> 여러분이 PC 앞에 없어도, 심지어 PC를 안 건드려도, 컴퓨터가 돌려놓습니다.

---

## Windows 작업 스케줄러가 뭔가요?

Windows에 **이미 내장된** 기능입니다. `taskschd.msc`를 실행하면 GUI가 뜹니다.

하는 일:
- "이 프로그램을 언제, 얼마나 자주 실행할지" 등록
- 등록하면 Windows가 그 시각마다 자동으로 실행
- PC가 꺼져있으면 다음 부팅 시 누락분을 실행하게 할 수도 있음

> 서버 관리자들이 '크론(cron)'이라고 부르는 것의 Windows 버전입니다.
> 특별한 프로그램을 설치할 필요 없이, Windows면 누구나 쓸 수 있습니다.

---

## 1단계: 왜 오전 11:30인가요?

Step 2에서 배웠듯이:
- 한국수출입은행 API는 **평일 오전 11:00 이후**에 그날 환율을 게시
- 10:59에 호출하면 빈 배열이 와서 "직전 영업일" 폴백이 발동 — 기술적으론 문제 없지만 **오늘 환율이 반영되는 건 내일**

그래서 **11:00에서 30분 여유**를 두고 11:30에 실행. 게시 지연·네트워크 지연까지 커버합니다.

> **교훈**: 스케줄 시각을 정할 때는 **데이터가 언제 준비되는지**부터 확인하세요.

---

## 2단계: AI에게 시키기

### Claude에게 입력

```
run_pipeline.py를 매일 오전 11:30에 자동 실행하는
Windows 작업 스케줄러 등록용 PowerShell 스크립트를 만들어줘.

요구사항:
- Task 이름: Week06Pipeline
- 실행 시각: 매일 11:30
- 절대경로 사용 (작업 스케줄러는 CWD가 달라서 상대경로 깨짐)
- 파이썬 실행 파일도 절대경로 (where py 로 찾아서)
- 스크립트 파일명: register_task.ps1
- 아직 실제 등록은 하지 말고 스크립트만 만들어줘
```

### 스크립트 내용 확인

Claude가 `register_task.ps1`을 만들어주면, 내용을 열어서 이런 구조인지 확인하세요:

```powershell
$TaskName  = "Week06Pipeline"
$PyPath    = "C:\Windows\py.exe"  # (실제 경로는 다를 수 있음)
$Script    = "C:\...\week06-hands-on\run_pipeline.py"
$WorkDir   = "C:\...\week06-hands-on"

$Action    = New-ScheduledTaskAction `
                -Execute $PyPath `
                -Argument $Script `
                -WorkingDirectory $WorkDir

$Trigger   = New-ScheduledTaskTrigger -Daily -At 11:30am

Register-ScheduledTask `
    -TaskName $TaskName `
    -Action $Action `
    -Trigger $Trigger `
    -Description "Week06 자동화 파이프라인 (매일 환율+매출 갱신)"
```

## 3단계: 직접 등록해보기

**직접 등록합니다.** 등록 → 확인 → 수동 실행 → 삭제까지 한 사이클을 체험합니다.

> 강의가 끝나면 반드시 삭제합니다 (5단계에서 안내). 삭제를 잊어도 `.env` 키가 본인 것이라 문제는 없지만, 불필요한 스케줄은 정리하는 게 좋습니다.

### 등록

Claude에게:

```
register_task.ps1을 만들어줘. 그리고 관리자 권한 PowerShell에서
실행할 수 있게 명령어를 알려줘.
```

> **관리자 권한 PowerShell 여는 법**: 시작 메뉴 → `PowerShell` 검색 → 검색 결과를 **우클릭** → **"관리자 권한으로 실행"** 클릭. Claude Code는 권한을 자동으로 높일 수 없으므로, 이 터미널에서 직접 명령을 붙여넣어야 합니다.

### 등록 확인

```
taskschd.msc를 띄우고 "작업 스케줄러 라이브러리"에서 Week06Pipeline을 찾아줘.
```

또는 PowerShell로:

```
Get-ScheduledTask -TaskName Week06Pipeline을 실행해줘.
```

## 4단계: 수동 실행으로 테스트

스케줄러에 등록한 작업을 **지금 당장** 한 번 돌려볼 수 있습니다. 11:30까지 기다릴 필요 없어요.

```
Start-ScheduledTask -TaskName Week06Pipeline을 실행하고,
logs/ 폴더의 가장 최근 파일을 보여줘.
```

기대:
- 스케줄러가 `run_pipeline.py`를 실행
- `logs/YYYYMMDD.log`에 실행 기록 append
- 성공 여부 확인

## 5단계: 등록 해제

등록 체험이 끝났으니 삭제합니다. **삭제도 자동화의 일부**입니다 — 안 쓰는 스케줄을 남겨두면 나중에 "이게 뭐였지?"가 됩니다.

```
Week06Pipeline 작업 스케줄러 등록을 해제해줘.
```

또는 관리자 PowerShell에서 직접:
```powershell
Unregister-ScheduledTask -TaskName Week06Pipeline -Confirm:$false
```

> **본인 업무에 적용할 때는 삭제하지 않습니다.** 오늘은 체험이니까 삭제하는 것이고, 실제로 매일 돌려야 하는 파이프라인이면 등록된 채로 두면 됩니다.

## 여기까지 하면 — 여러분이 가진 것

축하합니다. 여러분은 이제:

- 📥 **매일 자동으로** 환율과 매출 데이터를 DB에 반영하고
- 📝 **모든 실행 기록**을 로그로 남기며
- ⏰ **사람이 개입하지 않아도** 매일 오전 11:30에 돌아가는

**진짜 자동화 파이프라인**을 가지게 되었습니다.

week05에서 만든 대시보드는 **이제 매일 아침 출근하면 최신 데이터**로 갱신되어 있을 것입니다. 여러분이 아무것도 안 해도.

## 마지막 질문 — 여러분의 업무에 적용한다면?

잠시 생각해보세요:

1. 내 업무 중 **매달 / 매주 / 매일 똑같이 반복되는 것**이 있나요?
2. 그 작업의 **입력은 어디서** 오나요? (이메일? 공유폴더? 특정 시스템?)
3. 그 작업의 **출력은 어디로** 가나요? (엑셀? DB? 보고서? 다른 사람?)
4. 중간에 **사람의 판단**이 꼭 필요한 단계가 있나요? (있다면 그 단계 앞까지만 자동화)

이 질문의 답을 하나만 가지고 와서, 다음 주에 여러분만의 파이프라인을 만들어보세요.
