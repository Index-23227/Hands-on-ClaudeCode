## 파이프라인 = "스크립트를 줄줄이 엮기"

파이프라인이라는 말이 멋있어 보이지만 사실 별거 없습니다.

```
[스크립트 A]  →  [스크립트 B]  →  [스크립트 C]
 환율 받아오기     엑셀 → DB 반영    로그 남기기
```

그리고 이 전체를 **하나의 명령어**로 돌리는 것.

```
py run_pipeline.py
```

끝. 이거 하나가 내부에서 A → B → C를 순서대로 실행합니다.

그리고 마지막에:

> "이 명령어를 매일 오전 11:30에 컴퓨터가 알아서 돌려줘"

라고 Windows에게 시킵니다. 그러면 **여러분이 자는 동안에도 DB가 갱신됩니다.**

## 오늘 새로 배우는 개념 3개

### 1. 멱등성 (Idempotency)
> "같은 걸 몇 번 돌려도 결과가 같아야 한다"

**예**: 오늘 환율을 실수로 두 번 저장해도 DB에 2줄이 아니라 1줄만 있어야 한다.

재경팀에서 제일 무서운 실수: "저 혹시 이거 한 번 더 돌려도 돼요?" 멱등성이 지켜지면 → 돌려도 된다. 안 지켜지면 → 안 된다.

### 2. 외부 API (Application Programming Interface)
> "우리 DB 바깥의 데이터를 요청으로 받아오기"

지금까진 우리가 만든 엑셀·DB만 만졌습니다. 오늘은 **한국수출입은행 서버에 "오늘 환율 주세요"라고 요청**을 보냅니다.

현실 세계의 API는 **우리가 원하는 형식으로 주지 않습니다.** 오늘 그 함정을 몇 개 만나게 됩니다 (CLAUDE.md에 미리 다 적혀있음, Claude가 알아서 처리).

### 3. 스케줄 / 주기 실행
> "내가 PC 앞에 없어도 컴퓨터가 알아서 돌림"

Windows에는 **작업 스케줄러(Task Scheduler)**가 내장되어 있습니다. "매일 11:30에 이 파일 실행해" 하고 등록만 해두면 됩니다.
