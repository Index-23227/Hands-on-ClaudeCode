# 강사 리허설 가이드 — 수업 전 드라이런

> 수업 당일 당황하지 않으려면 **최소 한 번은 혼자 수업을 돌려봐야** 합니다.
> 이 문서는 그 리허설을 80~90분 안에 끝낼 수 있게 구조화합니다.
> 리허설에서 막힌 지점 = 수강생이 막힐 지점. 여기서 발견한 갭을 문서에 반영하고 수업에 들어가세요.

---

## 왜 이걸 해야 하나

- 10개 마크다운 문서가 있어도 **직접 해보지 않으면** 어디서 삐걱할지 모릅니다
- 데모가 실제로 되는지, 시간이 맞는지, 프롬프트가 의도대로 먹히는지는 **돌려봐야만** 알 수 있음
- 리허설 중 발견한 **3~5개 갭**을 고치면 수업 품질이 극적으로 올라감

## 소요 시간

- **빠른 버전**: 60분 (환경 10 + 쇼케이스 20 + 워크숍 20 + 복기 10)
- **충실한 버전**: 90분 (환경 10 + 쇼케이스 30 + 워크숍 40 + 복기 10)

초행이면 충실한 버전 추천.

(수업 시작 직전 한 번 더 정리.)

## Phase 3 — 워크숍 드라이런 (20~40분)

**목적**: 수강생이 막힐 지점을 본인이 먼저 체험하기.

### 3-1. 수강생 흉내 내기

강사로 말고 **초보 수강생**이 된 척:

1. 바탕화면에 `리허설테스트/` 폴더 만들기
2. [수강생과제_CLAUDE_템플릿.md](수강생과제_CLAUDE_템플릿.md) 복사해서 `CLAUDE.md`로 저장
3. Claude에게 **말로 설명**하며 빈칸 채우게 시키기:

```
내가 자동화하려는 업무를 설명할 테니 방금 만든 CLAUDE.md의
빈칸을 채워줘. 모르는 항목은 나한테 물어봐줘.
```

**주제**: [practice_cases/case1_경비집계.md](practice_cases/case1_경비집계.md) 또는 본인이 평소 하는 업무 중 하나.

### 3-2. 실제로 자동화 요청

CLAUDE.md 다 채우고 나서:

```
방금 채운 CLAUDE.md 규칙에 따라 [오늘 목표]를 만들어줘.
```

**체크**:
- [ ] 첫 결과가 몇 분 만에 나오나? (목표: 10분 이내)
- [ ] 결과가 CLAUDE.md의 요구(서식·정렬·색상)를 실제로 반영했나?
- [ ] 반영 안 된 부분을 **"거의 맞는데 X만 바꿔줘"** 패턴으로 고쳐봄 — 몇 회 반복해야 원하는 결과가 되나?

### 3-3. 검증체크리스트 돌리기

[검증체크리스트.md](검증체크리스트.md) 5개 실제로 수행:

- [ ] 행 개수 대조 — 몇 분 걸렸나?
- [ ] 합계 수기 계산 대조 — 차이가 있었나?
- [ ] 샘플 1건 수기 대조 — Claude 결과와 일치하나?
- [ ] 포맷 확인
- [ ] 예외 케이스 질문

**발견할 가능성이 큰 것**:
- 5개 전부 돌리면 15분 넘어감 → 수업 15분 검증 시간이 빡빡할 가능성
- Claude가 합계를 오반올림해서 ±몇 원 차이 날 수 있음 → 수강생이 "이거 틀린 건가?" 혼란

| # | 언제 막혔나 | 무엇이 문제였나 | 어느 문서로 풀리나 | 문서 업데이트 필요? |

### 4-2. 시간 실측

| 파트 | 계획 시간 | 실측 시간 | 차이 |
|---|---|---|---|
| 쇼케이스 | 30분 | ___분 | |

- **개선 권장** (여유되면): 프롬프트 예시 추가, 더 구체적 안내
- **다음 기수로** (당장 불필요): 큰 구조 변경

## 리허설 후 — 남은 3개 문서 판단

리허설 결과에 따라 **정말 필요한 것만** 만드세요.

### 🔴 프롬프트 원칙 1장 치트시트를 만들 필요가 있다면?

**다음 중 하나라도 해당되면 만드세요**:
- Phase 3에서 수정 프롬프트를 3회 이상 시도했는데 원하는 결과가 안 나옴
- 관찰 포인트(step0) + 막힘카드 + 검증체크를 동시에 펴놓고 보기 불편했음
- 수강생에게 "책상 옆에 A4 1장 붙여두세요"라고 말하고 싶어짐

### 🔴 시간 단축 시나리오를 만들 필요가 있다면?

**다음 중 하나라도 해당되면 만드세요**:
- Phase 2·3 합쳐서 90분을 넘겼음
- 어느 데모를 빼도 될지 판단이 빨리 안 섰음
- 실제 수업이 2시간 15분보다 짧을 가능성이 있음

### 🔴 수업 후 1주일 로드맵을 만들 필요가 있다면?

**다음 중 하나라도 해당되면 만드세요**:
- 리허설 중 "오늘 80%만 하고 나머지는 집에서"라고 말할 구간이 생김
- 수강생이 월·수·금 각각 무엇을 할지 구체 지시가 필요하다고 느낌
- 8주차 결과물 발표 수준이 수강생마다 너무 들쭉날쭉할 걱정이 듦

### 🟢 해당 없음 = 만들 필요 없음

전부 해당 없으면 **지금 있는 10개 문서가 실제로 충분한 것**입니다. 추가 문서는 오히려 혼란만 줍니다.

## 리허설 후 느낌이 "부족하다"면

가장 흔한 원인:
1. **개인 업무 예시가 준비 안 됨** — Phase 3에서 "내 업무"를 막연히 상상했다면, 수업 때도 수강생이 막힘. 구체적 예시 1~2개 미리 뽑아두기.
2. **시간 오버** — 위 "시간 단축 시나리오" 문서 필요
3. **문서 간 이동이 너무 많음** — 위 "프롬프트 1장 치트" 문서 필요

리허설 한 번 했는데도 불안하면 **두 번째 리허설**을 하세요. 두 번째는 30분이면 충분. 수업 품질 차이는 리허설 유무가 가장 큽니다.
