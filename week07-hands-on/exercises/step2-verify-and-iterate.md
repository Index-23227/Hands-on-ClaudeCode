# Step 2 — 검증과 반복: Claude를 의심하는 기술 (선택 심화)

> Step 1에서 "일단 돌아가는 결과물"을 만들었다면, 여기서는 그것을 **신뢰할 수 있게** 만듭니다.
> 비개발자에게 가장 중요한 스킬: **Claude가 틀린 것을 알아채는 법**.
> Step 1을 30분 안에 끝낸 수강생만 이 파트로 넘어옵니다.

---

## 왜 검증이 가장 중요한가

Claude는 **그럴듯한 틀린 답**을 만듭니다. 자주 있는 실패 유형:

| 틀리는 방식 | 어떻게 보이나 | 어떻게 잡나 |
|---|---|---|
| **엉뚱한 행을 빼먹음** | 결과 엑셀이 깔끔해 보이는데 원본보다 행 수 적음 | 행 개수 대조 |
| **단위를 섞음** | USD + JPY를 그냥 더해서 "매출 합계" 표시 | 통화 컬럼 확인 |
| **반올림 누적 오차** | 행별 반올림을 먼저 하고 합계 → 최대 8원 차이 | 원본 수기 합계와 대조 |
| **음수를 빠뜨림** | 환불/반품 행을 필터해버리고 매출만 집계 | 샘플 1~2건 직접 확인 |
| **NULL을 0 취급** | 데이터 미입력 법인을 "매출 0원"으로 표시 | 빈 값이 있어야 할 위치 확인 |

결과물이 보기 좋을수록 의심하세요. **Claude는 빨리 그럴듯해 보이게 만드는 데 최적화**되어 있습니다.

---

## 검증 루프 — 결과 받았으면 이 순서로

### 1회차: 표면 검증 (1분)

파일을 열어서 눈으로만:
- 열 제목이 내가 요청한 대로 있나?
- 행 수가 얼추 맞는 크기인가?
- 이상한 값(#REF!, ######, 빈 시트)이 있나?

### 2회차: 숫자 검증 (5분)

검증체크리스트 5개 (행 개수 / 합계 / 샘플 수기 / 포맷 / 예외).
특히 **샘플 1~2건 수기 대조**가 제일 강력합니다.

```
내가 수기로 계산해봤는데 ○○ 법인 4월 매출 원화 환산이
○원인데 결과는 ○원이야. 어디서 차이가 났어?
```

Claude가 이유를 설명하지 못하면 **의심 증거**입니다. 재계산 요청.

### 3회차: 예외 검증 (5분)

일부러 이상한 조건을 물어보세요:

```
결과에서 매출이 마이너스인 법인이 있어?
결과에서 값이 빈 칸인 법인이 있어?
다른 월에 비해 비정상적으로 크거나 작은 값이 있어?
```

Claude가 "없음"이라고 하면 원본에서 찾아지는지 **역으로** 확인:

```
원본 엑셀에서 매출이 마이너스인 행이 있으면 모두 보여줘.
```

양쪽이 맞아야 통과.

## 반복 프롬프트 패턴 — "거의 맞는데 X만"

### 패턴 1: 부분 수정
```
거의 맞는데 [부분]만 [이렇게] 바꿔줘. 나머지는 그대로.
```

### 패턴 2: 이유를 묻기
```
왜 [그 값]이 [그렇게] 나왔어? 계산 과정을 보여줘.
```

### 패턴 3: 대안 제시
```
지금 방식은 [문제]가 있어. [다른 방식]으로 해줘.
```

### 패턴 4: 제약 추가
```
[조건]을 만족해야 해. 이거 지키면서 다시 해줘.
```

### 하지 말 것 (초보자 함정)

- **"다시 해줘"만 말하기** — Claude가 같은 실수 반복
- **에러 보고 공황** — 에러는 복사해서 붙이면 대부분 풀림
- **완벽 추구** — 90% 맞으면 진행, 10%는 남겨두고 다음 주

## "이거 쓸 수 있나?" 최종 판단

업무에 쓸지 말지 결정하는 3가지 질문:

1. **재현 가능한가?** 같은 입력으로 두 번 돌리면 같은 결과가 나오나?
   → 지금 결과 파일 저장해두고, Claude에게 다시 돌리라고 해서 대조.

2. **책임질 수 있나?** 결과가 틀리면 내가 어디 가서 확인할 수 있나?
   → 원본 데이터 + Claude 대화 기록 + 내 수기 검증 3중 확인.

3. **설명할 수 있나?** 상사가 "이거 어떻게 만들었어?"라고 물으면 답할 수 있나?
   → CLAUDE.md + 실제 프롬프트를 스크린샷으로 가지고 있기.

3개 다 "예"면 업무에 쓰세요. 하나라도 "아니오"면 아직 연습.

## 한 수준 더 — "Claude에게 검증도 시키기"

사람 검증 + Claude 자체 검증 이중 체크:

```
방금 만든 결과물을 검증해줘. 다음을 확인하고 이상한 것 있으면 알려줘:
- 행 개수가 원본과 맞는지
- 통화 단위가 섞이지 않았는지
- 합계가 각 행의 합과 일치하는지
- 빈 값을 0으로 처리하지 않았는지
이상 없으면 "검증 통과"라고만 답하고, 있으면 항목별로 보고해줘.
```

Claude가 자신의 결과에서 오류를 찾으면 자동으로 수정 프롬프트도 요청하세요.

## 이 단계까지 하면

여러분은 이제 **"AI에게 시키는 사람"**에서 **"AI와 협업하는 사람"**으로 올라왔습니다.
이 차이를 8주차 결과물에서 보여주세요:
- 만든 것 (결과물)
- 검증한 방법 (체크리스트)
- Claude가 한 번 틀렸던 사례 + 어떻게 고쳤는지 (에피소드)

이 3가지가 들어가면 여러분의 최종 과제는 완성입니다.
