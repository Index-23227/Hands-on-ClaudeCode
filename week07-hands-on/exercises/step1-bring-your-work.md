# Step 1 — 본인 업무 개인지도 (각 15분, 2명 × 15분 = 30분)

> 쇼케이스 60분 관찰 끝. 이제 **여러분이 주인공**입니다.
> 쇼케이스에서 가장 공감됐던 데모 하나를 본인 업무에 접목합니다.
> 오늘 안에 완성할 필요 없음. **첫 한 단계**만.

> 📌 **두 분 모두 이미 본인 업무 폴더·CLAUDE.md·첫 task가 준비된 상태로 입장합니다.**
> 아래 1단계("설계")는 **확인·보완**만 하고 **2단계("구현")부터 집중**합니다.
> 1단계 내용은 참고용으로 유지 — 다음 달 새 업무에 재사용할 때 다시 볼 것.

---

## 흐름 (15분 × 2명)

```
┌─ 5분 ─┐    ┌─ 8분 ─┐    ┌─ 2분 ─┐
│ 설계   │ →  │ 구현   │ →  │ 검증   │
└───────┘    └───────┘    └───────┘
 CLAUDE.md    Claude에게    검증체크 5단계
 템플릿       자연어 요청    빠른 확인
 빈칸 채움    +수정 반복
```

**다른 분은 옆에서 관찰** — 같이 배우는 시간.

---

## 1단계: 설계 — CLAUDE.md 채우기 (5분)

### 1-1. 본인 업무 폴더

```
바탕화면/내업무자동화/
```

Claude에게 시켜도 OK: `바탕화면에 "내업무자동화" 폴더 만들어줘`

### 1-2. 템플릿 복사

```
week07-hands-on/수강생과제_CLAUDE_템플릿.md 내용을 그대로 복사해서
바탕화면/내업무자동화/ 폴더에 CLAUDE.md로 저장해줘.
```

### 1-3. 빈칸 채우기 — Claude에게 **말로**

타이핑 대신:

```
내가 자동화하려는 업무를 설명할 테니 방금 만든 CLAUDE.md의
빈칸을 채워줘. 모르는 항목은 나한테 하나씩 물어봐줘.

내 업무는: [한 문장]
지금 손으로 하는 과정은: [단계별 말로 설명]
가지고 있는 파일은: [엑셀인지, 몇 개인지, 어디]
```

**8번 "오늘 목표"까지**만 채우면 충분 (전체 다 채울 시간 없음).

### 🚨 민감정보 주의

실명·계좌·실제 거래처명이 있으면 **Claude에 올리기 전 익명화**:

```
이 엑셀에서 실명과 계좌번호를 가상 데이터로 바꿔줘.
구조는 유지한 채로. 익명화 파일을 "익명_원본이름.xlsx"로 저장.
```

---

## 2단계: 구현 — Claude에게 요청 (8분)

### 2-1. 첫 요청

오늘 공감 갔던 데모를 본인 업무에 접목:

**김인혜 과장님 예시** (F2 이상탐지 응용):
```
F2 데모(fallback_detect_and_send.py)를 참고해서 우리 회사 출고 데이터
(sample_shipments.xlsx, 익명화함)로 이상거래 탐지 기준을 변경해줘.
기준: [본인 회사 기준 설명]
결과: 담당자별 경각심 메일 초안 (실제 발송은 안 함)
```

**육아름 차장님 예시** (D 3종 출력 응용):
```
지금 Streamlit으로 만드는 Netra 대시보드 구조를 그대로 두고,
같은 데이터로 경영회의용 **PPT 2장**도 자동 생성되게 추가해줘.
월말마다 같은 프롬프트로 재실행하면 그달 수치가 들어간 PPT가 나오게.
```

### 2-2. Claude가 만들면 실행까지

`"이거 실행까지 해줘"` — 결과 파일 보여달라고.

### 2-3. 결과가 이상하면 — 수정 프롬프트

**처음부터 다시 X**. 쇼케이스에서 본 패턴:

```
거의 맞는데 [이 부분]만 [이렇게] 바꿔줘. 나머지는 그대로.
```

예:
- "합계 행이 빠졌어. 맨 아래에 넣어줘."
- "열 순서가 달라. [X]를 제일 왼쪽으로."
- "색깔 반대야. 증가를 빨강."

### 2-4. 막히면 — 막힘카드

3분 이상 못 풀면 [../막힘카드.md](../막힘카드.md):
- 에러 났으면 → **에러 통째로 Claude에 붙여넣기**
- 뭘 만들지 막연하면 → 5개 데모(A/B/D/F1/F2) 중 가까운 것 응용
- 그래도 3번 실패 → 강사 호출

## 3단계: 검증 — 신뢰 가능한가? (2분)

[../검증체크리스트.md](../검증체크리스트.md) 5단계 **빠르게**:

| 항목 | 빠른 확인 방법 |
|---|---|
| ① 행 개수 | 원본과 결과 행 수 눈으로 비교 |
| ② 합계 | 샘플 1개 수기 계산 |
| ③ 포맷 | 숫자·날짜 타입 맞나 |
| ④ 예외 | 빈 칸·음수 있으면 어떻게 처리됐나 |
| ⑤ 재현성 | 같은 입력으로 다시 돌리면 같은 결과? |

**15분 내 완벽 검증 불가능**. 이상하게 보이는 것 1~2개만 짚고 나머지는 숙제.

## 관찰하는 분 (옆사람)

다른 분 차례일 때 옆에서:
- 이 분 프롬프트 중 **내 업무에 쓸 만한 문장** 메모
- 막힘이 있었다면 **내 업무에 같은 막힘 있을지** 생각
- 결과물에서 **내 업무 규칙과 다른 점** 있는지

→ 8주차 발표 때 "다른 분 시연에서 배운 것" 한 줄 추가 가능.

---

## 마무리 (15분, 둘이 공유)

각자 1분씩 답:
1. 오늘 가장 인상 깊었던 데모 + 왜
2. 지금 만든 것을 다음 주까지 어떻게 완성할지
3. Claude가 틀렸을 때 가장 유용했던 수정 프롬프트

**8주차 예고**: 오늘 만든 첫 한 단계 + 1주일 동안 완성 + 5분 발표.
평가는 완성도가 아니라 **"Claude와 어떻게 대화했는가"**.
