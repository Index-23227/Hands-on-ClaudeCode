# 강사용 진단 카드 — 2인 개인지도 30분

> **수강생 2명**(김인혜·육아름) 15분씩 개인지도 시 진단용.
> 대규모 워크숍용 "순회"가 아니라 **1:1 코칭 시** 빠른 상황 판단 카드.

---

## 개인지도 15분 구조 (권장)

| 시점 | 내가 할 일 | 체크 |
|---|---|---|
| 0~3분 | 설계 단계 — CLAUDE.md 빈칸 채우기 옆에서 관찰 | 민감정보 있나? 오늘 목표가 3시간 스코프 넘나? |
| 3~8분 | 첫 프롬프트 → Claude가 만든 결과 확인 | 결과 나왔나? 에러인가? |
| 8~13분 | 수정 프롬프트 2~3회 반복 | "처음부터 다시" 패턴 빠졌나? 부분 수정 쓰나? |
| 13~15분 | 검증체크리스트 빠르게 + 남은 건 숙제 정리 | 80%면 OK. 완벽 추구하면 제지 |

## 상황별 처방 (다가가서 말할 한 줄)

### 상태 1: 시작 못 함
**신호**: "어… 어디서부터 해야 하죠?"
**처방**:
> "바탕화면에 '내업무자동화' 폴더 만드는 것부터.
> Claude에게 '바탕화면에 내업무자동화 폴더 만들어줘'로 시키세요. 그 다음 템플릿 복사."

### 상태 2: CLAUDE.md 빈칸 못 채움
**신호**: "이 항목을 뭘 써야 할지…"
**처방**:
> "저한테 한 번 말로 설명해 주세요. 그걸 그대로 Claude에게 붙여넣으세요.
> 정리는 Claude가 합니다."

### 상태 3: 스코프 과욕 (3시간 초과)
**신호**: "월 결산 전체를 자동화하고 싶어요" / "40개 법인 전부 한 번에"
**처방**:
> "오늘은 첫 한 단계만. 그중 가장 먼저 하는 일이 뭐예요?
> [그 하나]를 8번 '오늘 목표'에 고정. 나머지는 숙제."

### 상태 4: 에러 반복
**신호**: Claude 응답에 같은 에러 3회 이상
**처방**:
> 1. 에러 메시지 **전체**를 복사해서 Claude에게 붙였나 확인
> 2. 안 했으면: "에러 그대로 붙여넣고 '이거 해결해줘' 하세요"
> 3. 해도 반복이면 → 프롬프트가 모호 → CLAUDE.md 해당 섹션 더 구체적으로
> 4. 환경 문제 (패키지·경로) 의심되면 직접 진단 (아래 "환경 진단")

### 상태 5: 결과 미묘하게 이상
**신호**: "결과가 90%는 맞는데 이 부분이…"
**처방**:
> **"거의 맞는데 [X]만 [Y]로 바꿔줘. 나머지는 그대로"** 패턴 알려주기.
> 이게 **오늘의 진짜 스킬**. 쇼케이스에서도 강조했던 것.

### 상태 6: 완벽주의
**신호**: 색상·폰트 조정에 5분 이상
**처방**:
> "지금 80% 맞으면 검증체크리스트 먼저 돌리세요.
> 다듬기는 숙제. 다음 주에 완성하면 됩니다."

## 환경 진단 (강사가 직접 해결해야 할 것)

| 증상 | 원인 | 1분 해결 |
|---|---|---|
| `ModuleNotFoundError: openpyxl` | 패키지 미설치 | Claude에게 `"openpyxl 설치해줘"` |
| `ModuleNotFoundError: pptx` | python-pptx 미설치 | `py -m pip install python-pptx` |
| `FileNotFoundError` + 한글 경로 | Windows 경로 인코딩 | 절대경로로 바꾸거나 영문 폴더 |
| `PermissionError: file being used` | 결과 엑셀이 열려있음 | 엑셀 창 닫고 재실행 |
| `UnicodeEncodeError 'cp949'` | 콘솔 인코딩 | 스크립트 상단에 `sys.stdout.reconfigure(encoding="utf-8")` (대부분 prep 스크립트엔 이미 있음) |
| `535 BadCredentials` (SMTP) | 앱 비밀번호 오류 | [Gmail_자동화_설정.md](Gmail_자동화_설정.md) 가이드 |
| Python 자체 안 켜짐 | PATH 문제 | `py --version` 먼저 체크 |

**원칙**: 환경 문제는 강사가 즉시 해결. 수강생에게 맡기면 15분 허비.

## 두 분 업무별 예상 시나리오

### 김인혜 과장님 (F2 이상탐지 응용)

**예상 목표**: 본인 회사 SAP 출고 데이터로 이상거래 탐지 + 담당자 메일 초안

**예상 막힘**:
- 실제 SAP 컬럼 이름이 `shipments.xlsx`와 다름 → Claude에게 "컬럼명 이렇게 매핑해줘" 가이드
- 이상 기준이 우리(Fixture 50) vs 본인(?) 다름 → "우리 회사 기준은 [X]" 설명 프롬프트
- Outlook 자동화도 원할 수 있음 → SMTP 대신 pywin32 Outlook 설명

**검증 포인트**:
- 탐지 건수가 상식적인 범위인지
- 메일 수신자가 "본인 부계정"으로만 되어있는지 (안전장치)

### 육아름 차장님 (D 3종 출력 or F1 응용)

**예상 목표 1 (D)**: Netra 대시보드 데이터로 PPT 월간보고 자동 생성
**예상 목표 2 (F1)**: 40법인 회신 메일 취합 (MCP로 본문 파싱 + 필요시 IMAP으로 첨부까지)

**예상 막힘 (D)**:
- 실 Netra 컬럼 구조가 `overseas_sales.xlsx`와 다름
- PPT 브랜드 템플릿 요구 (회사 로고·색) → "템플릿 pptx 파일이 있으면 그걸 기반으로"
- 40법인 너무 많아 PPT 슬라이드 폭주 → "상위 10개 + 지역별 요약"으로 축소 유도

**예상 막힘 (F1)**:
- 학교메일(yonsei.ac.kr) 회사 정책으로 IMAP 막혔으면 → 개인 Gmail로 전환
- 메일 제목 패턴이 다양 → 키워드 여러 개 OR 조건

**검증 포인트**:
- PPT 차트의 숫자와 원본 xlsx 합계 일치
- 누락 법인 리스트 정확
