# 강사용 순회 체크리스트

> 90분 워크숍 동안 강사가 수강생 사이를 돌며 빠르게 진단하기 위한 카드.
> 수강생 한 명당 30초~2분. **모두에게 공평하게 가기보다, 막힌 사람을 먼저**.

---

## 순회 타이밍

| 시점 | 무엇을 확인 |
|---|---|
| 워크숍 시작 후 **5분** | CLAUDE.md 템플릿 열었나, 본인 업무를 말로 설명하고 있나 |
| 워크숍 시작 후 **20분** | CLAUDE.md 다 채웠나, Claude에게 첫 요청 보냈나 |
| 워크숍 시작 후 **45분** | 결과물이 한 번이라도 나왔나 (완벽 불필요) |
| 워크숍 시작 후 **70분** | 검증체크리스트 돌고 있나 |

각 시점에서 **뒤쳐진 사람을 우선 발견**. 완벽하게 진행 중인 사람은 방해하지 마세요.

## 막힘 신호 (멀리서 보고도 식별)

아래 중 하나라도 보이면 **우선 다가갈 대상**:

- [ ] 5분 이상 터미널/에디터만 보고 있고 타이핑이 없음

- [ ] 노트북을 덮으려고 함 (포기 직전)

## 수강생 상태 5단계 진단

다가갔을 때 **"지금 어디까지 왔어요?"** 한 마디로 시작. 답에 따라 분류:

### 상태 1: 시작 못 함 (폴더/템플릿 안 만듦)

- **증상**: "어… 어디서부터 해야 하죠?"
- **처방**:
  ```
  "일단 바탕화면에 '내업무자동화' 폴더 하나 만드세요. Claude한테
  '바탕화면에 내업무자동화 폴더 만들어줘'라고 시키시면 돼요.
  다음에 템플릿을 복사하는데, 그것도 Claude한테 시키세요."
  ```
- **보조 문서**: [exercises/step1-bring-your-work.md](exercises/step1-bring-your-work.md) 1단계

### 상태 2: CLAUDE.md 채우다 막힘

- **증상**: "이 항목을 뭘 써야 할지 모르겠어요"
- **처방**:
  - 그 항목을 **사람 말로** 강사에게 설명시킴
  - 수강생이 말한 그대로 Claude에게 옮겨 적게 함
  ```
  "방금 저한테 설명하신 내용을 그대로 Claude에게 붙여넣으세요.
  정리는 Claude가 해줍니다."
  ```

### 상태 3: 스코프 폭탄 (과욕)

- **증상**: "저는 월결산 전체를 자동화하려고 해요"
- **처방**:
  ```
  "오늘은 그중 첫 한 단계만. 월결산에서 제일 먼저 하는 일이 뭐예요?"
  ```
  → 그 하나를 CLAUDE.md 8번 "오늘 목표"에 고정. 나머지는 숙제.

### 상태 4: 에러 반복 / 결과가 계속 이상

- **증상**: "Claude가 계속 엉뚱하게 만들어요"
- **처방 순서**:
  1. 에러나 이상한 결과를 **그대로 복사**해서 Claude에게 붙여넣었는지 확인
  2. 안 했으면 지금 시킴: `에러 그대로 복붙 + "이거 해결해줘"`
  3. 그래도 반복되면 **프롬프트가 모호**한 것 — CLAUDE.md 해당 섹션을 더 구체적으로 다시 쓰게
  4. 환경 문제(패키지 미설치, 경로 오류) 의심되면 강사가 직접 진단 (아래 "환경 진단" 참고)

### 상태 5: 완벽주의 (결과 나왔는데 계속 다듬음)

- **증상**: "색깔이 미묘하게 달라서요…"
- **처방**:
  ```
  "지금 80% 맞으면 다음 단계로 가세요. 검증체크리스트 먼저 돌리고,
  다듬기는 숙제로. 오늘 안 끝내도 괜찮아요."
  ```
  → 검증체크리스트 문서로 넘김.

## 환경 진단 (강사가 직접 풀어줘야 하는 것)

수강생이 해결 못 하는 영역. 강사가 빠르게 진단해야 함.

| 증상 | 원인 | 1분 해결 |
|---|---|---|
| `ModuleNotFoundError: No module named 'openpyxl'` | 패키지 미설치 | Claude에게: "openpyxl 설치해줘" |
| `FileNotFoundError` + 경로에 한글 | Windows 경로 인코딩 | 절대경로로 바꾸거나 영문 폴더로 이동 |
| `PermissionError: file is being used` | 결과 엑셀이 열려있음 | 엑셀 창 닫고 재실행 |
| `UnicodeDecodeError` | 파일 인코딩(`cp949` vs `utf-8`) | Claude에게: "`encoding='cp949'`로 열어줘" |
| `sqlite3.OperationalError: unable to open database file` | week05 DB 경로 문제 | `../week05-hands-on/data/sales.db` 존재 확인 |
| Python 자체가 안 켜짐 | PATH 문제 | `py --version` 체크 후 안내 |

**원칙**: 환경 문제는 강사가 즉시 해결해줘야 합니다. 수강생에게 맡기면 20분 허비.

## 마무리 공유 진행 (워크숍 마지막 15분)

1~2명만 발표. 선정 기준:

- ✅ **막혔다가 푼 경험이 있는 사람** 우선 (학습 효과가 큼)
- ✅ 보통 수준의 완성도 (너무 잘한 사람은 위축 효과)
- ❌ 완벽하게 끝낸 사람은 굳이 발표 안 시켜도 됨

발표자에게 던질 질문 3개:
1. "어디서 가장 많이 막혔어요?"
2. "Claude에게 처음 말했을 때랑 나중이랑 말하는 방식이 달라졌나요?"
3. "8주차에는 이걸 어떻게 완성할 거예요?"

마지막 3번이 **8주차 과제로 연결**되는 핵심 질문.

## 끝나고 혼자 복기할 것

수업 후 강사가 메모할 것 (다음 회차 개선용):

- 어느 단계에서 가장 많이 막혔나 (공통 패턴)
- 예상 못 한 에러가 있었나
- 공용 연습 케이스 3개 중 어느 게 인기였나
- 시간 배분이 맞았나 (30/90/15)

이 메모가 **8주차 준비**와 **다음 기수 커리큘럼 개선**의 재료.
