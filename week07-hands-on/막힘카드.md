# 막힘 카드 — 이럴 땐 이렇게

> 막혔다고 손 들기 전에 이 카드 먼저 보세요.
> 70%는 여기서 풀립니다. 안 풀리면 강사 호출.

---

## 1. 빨간 글씨(에러)가 떴어요

**하지 말 것**: 얼어붙지 마세요. 에러는 Claude가 읽으면 대부분 풉니다.

**이렇게 하세요**:
1. 에러 메시지를 **처음부터 끝까지** 드래그해서 복사 (Traceback 전체)
2. Claude에게 그대로 붙여넣고 이 한 줄:

```
위 에러 해결해줘. 원인이 뭐고 어떻게 고치면 돼?
```

3. Claude가 수정안을 주면 "그럼 그렇게 고쳐서 다시 실행해줘"

**그래도 안 되면**: 같은 에러가 3번 반복되면 강사 호출.

---

## 2. 내가 하고 싶은 게 어떤 유형인지 모르겠어요

오늘 본 **쇼케이스 5종** 중 가까운 것을 찾아 프롬프트 응용:

| 내 상황 | 가까운 데모 | 프롬프트 힌트 |
|---|---|---|
| 이미지·사진에서 숫자·항목 뽑고 싶음 | **A** 영수증 OCR | "이 사진들에서 날짜·항목·금액 뽑아 엑셀로" (이미지 첨부) |
| 외부 웹·API에서 데이터 가져와 요약 | **B** DART | "○○ API에서 ○○ 가져와서 비교표 만들고 3줄 요약" |
| 같은 데이터를 여러 포맷(PPT/HTML/React)으로 | **D** 3종 대시보드 | "이 xlsx로 PPT 만들고, 같은 내용 HTML로도, React 컴포넌트로도" |
| 받은편지함 메일 조회·본문 취합 | **F1** 메일 수신 | "내 받은편지함에서 '○○ 보고' 제목 메일 본문에서 숫자 뽑아 취합" (MCP Gmail) |
| 받은편지함 **첨부 파일**까지 저장/취합 | **F1 IMAP 폴백** | "IMAP으로 연결해서 '○○' 제목 메일 첨부파일 다운로드 후 통합" (.env 필요) |
| 조건에 맞는 대상자에게 개인화 메일 발송 | **F2** 이상거래 알림 | "○○ 데이터에서 조건에 맞는 ○○ 찾아 담당자별 개인화 메일 SMTP 발송" |

각 데모 상세 프롬프트는 `demos/{폴더}/README.md`에 있음 (참고: `demos/receipt_ocr`, `demos/dart_samsung`, `demos/dashboard_3format`, `demos/email_receive`, `demos/email_send`).

## 3. Claude가 엉뚱한 걸 만들었어요

**처음부터 다시 시키지 마세요.** 90% 맞을 땐 나머지 10%만 고치면 됩니다.

**이렇게 말하세요**:
```
거의 맞는데 ○○ 부분만 ○○로 바꿔줘. 나머지는 그대로.
```

예시:
- "거의 맞는데 금액 컬럼 숫자에 천단위 콤마만 넣어줘"
- "좋은데 합계 행이 빠졌어. 맨 아래에 합계 넣어줘"
- "색깔 반대야. 증가를 빨강, 감소를 초록으로 바꿔줘"

---

## 4. 엑셀 구조가 복잡해서 Claude가 못 읽어요

병합셀·색상·수식이 많으면 Claude가 "데이터가 어딘지 모르겠다" 상태가 됩니다.

**이렇게 시작하세요**:
```
이 엑셀의 구조부터 알려줘. 어떤 시트에 어떤 컬럼이 어디부터 있어?
```

구조를 파악한 다음 원래 하려던 작업을 시키세요.

---

## 5. 내 데이터를 그대로 올려도 되나 고민돼요

**민감정보(실명·실계좌·실주민번호 등)는 올리지 마세요.**

**이렇게 하세요**:
```
이 엑셀의 법인명·계좌번호·금액 같은 민감정보를 가상 데이터로 바꿔줘.
구조는 유지한 채로 익명화해서 저장해줘.
```

익명화된 파일로 실습하세요.

---

## 6. 강사 호출 타이밍

아래 중 하나면 손 드세요:

- [ ] 위 1~5 해봤는데 같은 에러 3번 반복
- [ ] 파이썬/패키지 설치 관련 에러 (경로·권한·PATH 메시지)
- [ ] 내 업무가 오늘 3시간 안에 끝날 수 있는 스코프인지 판단 안 됨
- [ ] 결과물이 맞는지 틀린지 판별이 안 됨 → **검증 체크리스트** 먼저 돌려보세요
