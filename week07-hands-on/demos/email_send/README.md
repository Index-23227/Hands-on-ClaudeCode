# 데모 F2 — 이상거래처 자동 탐지 + 담당자별 알림 메일 (IMAP/SMTP)

> **수강생이 느낄 것**: "이상 감지에서 경각심 메일까지 1분, 담당자마다 다른 내용으로"
> **핵심 메시지**: F1은 대화형(MCP), F2는 **주기 실행 자동 파이프라인** (IMAP/SMTP).
> **시연 시간**: 12분

---

## 업무 맥락 (김인혜 과장님 직결)

매주 이상출고 탐지 → 해당 거래처 담당자에게 경각심 메일 알림.
지금은 SAP 데이터 수기 조회 → 엑셀 분석 → 메일 한 통씩 작성.
이 업무를 **스크립트 한 번 실행**으로 끝내는 쇼케이스.

> **오늘 시연은 강사 Gmail → 강사 부계정 3~5개**로만 발송.
> 실명·실제 거래처 이메일은 절대 시연 중 등장 안 함.

---

## 이상 탐지 기준 (1조 문서에서 가져옴)

| 조건 | 이상도 | 의미 |
|---|---|---|
| 품목별 50개 초과 출고 | **이상 품목** 식별 | 과출고 의심 |
| 이상 품목 2개 이상 **+ 반품률 0%** | **높음** | 가공매출 강력 의심 |
| 이상 품목 있음 + **반품률 0%** | **주의** | 반품 패턴 비정상 |
| 이상 품목 있음 + **반품률 15% 초과** | **주의** | 부도·허위출고 의심 |
| 그 외 | 정상범위 | 일상적 거래 |

Mock 데이터(`data/shipments.xlsx`)에 **5곳의 이상거래처** 내장:
- CUST001 강남B치과 [높음] — 이상품목 3개, 반품 0%
- CUST002 판교Y치과 [높음] — 이상품목 2개, 반품 0%
- CUST003 잠실Z치과 [주의] — 반품 20%
- CUST005 홍대M치과 [주의] — 반품 0%
- CUST004 성수K치과 [정상범위] — 반품 4% (제외 대상)

→ 실제 발송: **4곳** (CUST004는 정상범위라 제외)

## 시연 흐름

### 사전 준비 (강사)

#### 1. .env 설정
```

GMAIL_APP_PASSWORD=xxxx xxxx xxxx xxxx

```

#### 2. 이상거래 Mock 데이터
```bash
py week07-hands-on/prep/make_abnormal_shipments.py
```

---

### 시연 프롬프트 1 — 탐지

```
week07-hands-on/demos/email_send/data/shipments.xlsx를 읽어서
이상 거래처를 탐지해줘.

기준:
- 품목별 출고수량 > 50개 = "이상 품목"
- 이상 품목 개수 & 반품률(반품수량/출고수량)로 등급 분류:
  - 이상품목 2개 이상 + 반품률 0% → "높음"
  - 이상품목 있음 + 반품률 0% → "주의"
  - 이상품목 있음 + 반품률 > 15% → "주의"
  - 그 외 → "정상범위" (알림 대상 아님)

탐지 결과를 표로 보여줘: 거래처코드/거래처명/등급/이상품목수/반품률/담당자/담당자이메일
```

### 시연 프롬프트 2 — 메일 초안 생성 (미발송)

```
위 탐지 결과에서 등급이 "높음" 또는 "주의"인 거래처를 대상으로,
각 담당자에게 보낼 경각심 메일 초안을 개인화해서 작성해줘.

요구사항:
- 제목: "[가공매출 의심] {거래처명} 출고 패턴 점검 요청 (2026-03)"
- 본문:
  - 담당자명 + 정중한 인사
  - 해당 거래처의 이상 패턴 구체 기술 (이상품목·출고수량·반품률)
  - "영업 팀 차원 확인 후 회신 부탁드립니다" 문구
  - 서명: 재경팀 (강사 이름)
- 톤: 공손하지만 사안 심각성 전달

거래처별로 다른 내용이 들어가야 함.
실제 발송은 하지 말고 각 초안을 화면에 모두 출력해줘.
```

### 시연 프롬프트 3 — 실제 발송

```
초안이 OK. 이제 .env의 SUB_ACCOUNTS로 실제 발송해줘.
- shipments.xlsx의 담당자이메일 컬럼에 있는 SUB1/SUB2/SUB3 라벨을
  .env의 SUB_ACCOUNTS 순서대로 매핑
- SMTP로 발송 (smtp.gmail.com:465)
- 발송 성공/실패 로그 출력

⚠️ 발송 전 한번 더 "이 주소로 보낼게요 — 맞나요?" 확인해줘.
```

Claude가 최종 확인받고 SMTP 실행 → 부계정 받은편지함에서 실제 메일 확인.

## 수강생에게 강조할 포인트

### 1. **"스크립트가 무인으로 돌 수 있다"**
- 김인혜 과장님이 매주 수요일 09시 자동 실행 → 이상 탐지 → 알림 발송
- Windows 작업 스케줄러(week06에서 배움)에 등록하면 완성

### 2. **개인화의 자유**
- 담당자마다 다른 문장, 다른 거래처 정보
- 15곳 거래처에 각각 다른 내용 = 1분

### 3. **F1과의 차이 — 언제 뭘 써야 하나**
| | F1 (MCP 대화) | F2 (IMAP/SMTP 스크립트) |
|---|---|---|
| 시점 | 강사 임시 조회 | 매주 수요일 무인 실행 |
| 사용자 | 육아름 차장 (조회) | 김인혜 과장 (자동 탐지) |
| 조작 | "메일 봐줘" 한 마디 | 스크립트 한 번 등록 |

### 4. **안전장치**
- 자동 발송 전 항상 사람이 **초안 확인**
- 부계정 테스트 → 실제 거래처로 확장은 별도 승인
- 수신자 주소 로그 남김 (감사 대응)
