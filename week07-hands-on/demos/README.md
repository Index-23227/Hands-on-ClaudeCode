# demos/ — 강사용 폴백 정답 스크립트

> 7주차 쇼케이스에서 **Claude가 즉흥 생성에 실패하거나, 시간이 부족할 때** 사용하는 참고 정답.
> **수강생에게 직접 보여주지 마세요.** 수강생은 강사의 프롬프트와 Claude의 생성 과정을 봐야 합니다.
> 정답 파일을 먼저 열면 "이미 있는 걸 Claude가 만드는 척한다"는 오해가 생깁니다.

## 언제 이 파일을 쓸까

### 시나리오 A — Claude가 즉흥 생성에 실패했을 때
증상:
- Claude가 3~4번 시도해도 결과물이 엉망
- 에러가 반복되며 시간 초과
- 수강생 주의 집중력이 떨어지고 있음

대응:
```
"시간상 미리 준비한 답을 보여드릴게요"
```
라고 솔직히 말한 뒤:
```bash
py demos/demo1_compare_excel.py
```

생성된 결과를 열어서 "이게 여러분이 Claude에게 시킨 결과와 같은 모양입니다"로 연결.

### 시나리오 B — 시간이 15분 이상 부족할 때
6개 데모 중 2~3개는 폴백 실행으로 건너뛰고 결과만 보여주기. 쇼케이스의 핵심은 **프롬프트 패턴**이므로 전부 라이브로 하지 않아도 됨.

### 시나리오 C — 수업 후 복습 요청
수강생이 "집에서 다시 보고 싶다" 하면 이 스크립트를 직접 돌려보라고 안내 (프롬프트 → 코드 → 결과의 흐름을 혼자 관찰).

## 수강생과의 차이 — 왜 이 코드는 Claude가 만든 것과 다를 수 있나

이 스크립트는 **강사 사전 테스트를 통과한 안정적 버전**입니다. Claude가 실시간으로 만드는 코드와 비교하면:

| 항목 | 이 폴백 스크립트 | Claude 즉흥 생성 |
|---|---|---|
| 에러 처리 | 경로/의존성을 미리 검증 | 종종 빠뜨림 |
| 출력 경로 | `HERE/output/` 절대 경로 고정 | 가끔 현재 CWD에 저장 |
| 스타일 | 수업 한 번만 쓰기 좋게 간결 | 더 복잡하거나 더 단순할 수 있음 |
| 재현성 | 몇 번 돌려도 같은 결과 | 프롬프트 표현에 따라 달라짐 |

**수강생에게 설명할 때**: "Claude가 만든 건 이것보다 더 길거나 짧을 수 있어요. 중요한 건 결과가 같냐입니다."
