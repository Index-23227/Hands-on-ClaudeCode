# 데모 A — 영수증 사진 → 경비 엑셀 (Claude Vision 멀티모달)

> **수강생이 느낄 것**: "사진이 엑셀이 되네. 영수증 정리에 매달 반나절 쓰던 게 끝났다"
> **핵심 메시지**: Claude Code는 텍스트뿐 아니라 **이미지도 직접 읽는다**. 별도 OCR 라이브러리 불필요.
> **시연 시간**: 10~12분

---

## 시연 흐름

### 1단계 — 이미지를 Claude에게 "첨부"

`samples/` 폴더의 영수증 PNG 8장을 **Claude Code 채팅창에 드래그앤드롭** 또는 파일 경로로 지정.

### 2단계 — 프롬프트 (그대로 복붙)

```
week07-hands-on/demos/receipt_ocr/samples/ 폴더의 영수증 8장을 모두 읽어서,
각 영수증에서 아래 정보를 추출해줘:
- 날짜
- 매장명
- 항목 요약 (대표 품목 1~2개)
- 합계 금액
- 결제방식 (법인카드 / 개인카드)

추출한 결과를 "경비정리_2026-03.xlsx"로 정리해줘. 다음 규칙 적용:
- 시트1: 전체 목록 (8행) — 날짜순 정렬
- 시트2: 결제방식별 분리 (법인카드 섹션 / 개인카드 섹션), 각 섹션 마지막에 합계 행
- 헤더는 짙은 남색 배경 + 흰 글씨
- 합계 행은 회색 배경 + 볼드
- 금액은 천단위 콤마
- 저장 위치: week07-hands-on/demos/receipt_ocr/output/
```

### 3단계 — 결과 확인

- `output/경비정리_2026-03.xlsx` 열기
- 시트1에서 8행 모두 추출됐는지
- 시트2에서 법인카드/개인카드 분리·합계 정확한지

### 기대 결과 (검증용)

| 결제방식 | 건수 | 합계 |
|---|---|---|
| 법인카드 | 6건 | 561,300원 |
| 개인카드 | 2건 | 52,900원 |
| **전체** | **8건** | **614,200원** |

## 수강생에게 강조할 포인트

1. **"OCR 라이브러리 설치 안 했습니다"** — Claude가 이미지를 직접 봄
2. **"저는 엑셀 구조를 말로만 설명했습니다"** — 헤더·정렬·합계 전부 자연어로
3. **"이걸 매달 반복하려면?"** — 다음 달엔 같은 프롬프트 + 새 사진만 바꾸면 끝

## 확장 아이디어 (시간 남으면)

### 영수증 추가 추출
```
각 영수증에서 사업자번호와 승인번호도 추가로 뽑아줘. 감사 대응 자료로 쓸거야.
```

### 실제 사진으로 전환
강사가 본인 폰으로 찍은 실제 영수증 사진을 업로드:
```
이건 제가 방금 찍은 실제 영수증이야. 여기서도 같은 방식으로 추출 가능해?
```
→ 실사진이 OK되는 걸 보여주면 즉각 "나도 할 수 있다" 반응.

### 경비 분류
```
이 8건을 교통비 / 식대 / 사무용품 / 출장비 / 접대비로 자동 분류해서
분류별 합계도 함께 정리해줘.
```

---

## 폴백 정답 (시연 실패 시 강사용)

Claude Vision이 작동 안 하거나 시간 부족 시:

```bash
py week07-hands-on/demos/receipt_ocr/fallback_extract.py
```

→ `output/경비정리_2026-03.xlsx`가 동일한 형식으로 즉시 생성됨.
수강생에게 "이게 Claude가 만든 결과입니다"로 설명 (정직하게 "미리 준비한 참고 정답"이라고 말해도 OK).

---

## 사전 준비 (강사)

1. `prep/make_receipt_samples.py` 한 번 실행 → `samples/` 에 PNG 8장 있어야 함
2. Claude Code가 이미지 첨부 기능 지원 여부 확인 (드래그앤드롭 or `@파일경로`)
3. 리허설 시 프롬프트 한 번 실행해서 결과 확인
