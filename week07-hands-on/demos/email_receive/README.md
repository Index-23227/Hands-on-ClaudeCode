# 데모 F1 — 법인 회신 메일 자동 취합 (MCP Gmail)

> **수강생이 느낄 것**: "받은편지함 여는 것부터 첨부 열기·복사·붙여넣기까지 자동이네"
> **핵심 메시지**: Claude는 내 Gmail에 **직접** 들어갈 수 있다 (MCP).
> 받은편지함 읽기·분류·첨부 취합이 **한 번의 대화**로 끝난다.
> **시연 시간**: 12~14분

---

## 업무 맥락 (육아름 차장님 직결)

매달 40개 해외법인 담당자가 월말 매출 보고를 **이메일로** 회신.
지금은 40통을 **한 통씩 열어** 첨부 다운로드 → 폴더 정리 → 엑셀 취합 → DB 반영.
이 업무 반나절을 **한 프롬프트**로 끝내는 쇼케이스.

> **오늘 시연은 8법인(강사 Gmail 자체 발송 메일)으로 축소**.
> 40법인 확장도 동일 구조 — "이름만 늘어날 뿐 로직은 같음" 강조.

---

## 시연 흐름

### 사전 준비 (강사)

#### 1. 샘플 메일 발송 (수업 1~2시간 전)
```bash
py week07-hands-on/prep/send_sample_inbox_emails.py
```
→ 강사 Gmail 받은편지함에 8통의 "법인 담당자 매출 보고" 도착

#### 2. Gmail MCP 연결 (1회, 수업 이전)

**Claude.ai 네이티브 Gmail 커넥터**를 쓴다 (가장 간단).

1. Claude Code 또는 claude.ai에서 Settings → Connectors → Gmail 활성화
2. OAuth 로그인 → 강사 본인 Gmail 계정 승인
3. 툴 이름 `mcp__claude_ai_Gmail__*` 가 자동으로 사용 가능해짐 (별도 `.mcp.json` 불필요)

**검증**: 프롬프트에 "내 Gmail에서 제목에 '매출 보고' 포함된 메일 찾아줘" → 목록 뜨면 OK.

> ⚠️ **`claude_ai_Gmail`의 제약**: 이 커넥터는 `search_threads` / `get_thread` 는 되지만
> **첨부 바이너리 다운로드는 제공하지 않는다** (2026-04 기준).
>
> 해결책: 샘플 메일 본문에 이미 `매출액: 101,000 EUR` 형식으로 금액이 박혀 있으므로
> **본문 파싱만으로 8법인 데이터 취합이 가능**하다. 첨부 다운로드가 필요하면 IMAP 폴백.

설정이 막히면 아래 "폴백 실행"으로 진행.

### 시연 프롬프트 1 — 조회

```
내 Gmail 받은편지함에서 제목이 "매출 보고"를 포함하는 최근 메일을 찾아서
보낸 날짜·제목·첨부파일명을 목록으로 보여줘.
```

**기대 결과**: 8통이 표로 나열됨 (CN01, DE01, GB01, IN01, JP01, TH01, US01, VN01 순)

**수강생 반응 지점**: "저 메일들 전부 Claude가 직접 본 거네?"

### 시연 프롬프트 2 — 본문 파싱 + 환율 환산

```
위 8통의 메일 본문에서 각 법인의 통화·매출액·담당자를 추출하고
한 엑셀로 취합해줘.

요구사항:
- 메일 본문에 "매출액: 875,000 CNY" 형식으로 적혀 있음. 정규식 파싱.
- 원화(KRW) 환산은 week05 DB(../week05-hands-on/data/sales.db)의
  exchange_rates에서 **2026-03-31 월말 환율**을 조인해서 사용
  (MAX(rate_date) 말고 정확히 '2026-03-31'로 필터)
- 법인명 가나다순 정렬 (독일→미국→베트남→영국→인도→일본→중국→태국)
- 저장: week07-hands-on/demos/email_receive/output/매출취합_2026-03.xlsx
  - 시트1 "전체": 법인코드·법인명·통화·현지금액·환율(KRW)·원화환산(KRW)·담당자·비고
  - 합계 행 (KRW 합계 = 1,212,620,000 예상)
  - 헤더 짙은 남색 배경(#1F4E78) + 흰색 볼드
  - 숫자 우측정렬·천단위 콤마·틀고정(A2)
  - 시트2 "누락": 기대 8법인 중 회신 없는 법인 (지금은 전원 회신 → "— 누락 없음 —")
```

**시연 팁**: Claude가 "첨부를 다운로드하겠다"고 하면 **"본문만 파싱해"** 로 가볍게 수정.
MCP 커넥터가 첨부를 안 주는 제약을 자연스럽게 에둘러가는 흐름으로 보여주기.

### 시연 프롬프트 3 (선택) — 누락 확인·후속 조치

> ⏱️ 시간 남을 때만. 기본 샘플(8/8 수신)로는 "누락 없음" 결과만 나와서 임팩트 약함.
> 강하게 보여주고 싶으면 사전에 `send_sample_inbox_emails.py --count 6` 으로 일부만 발송해서 2법인 누락 상태로 시연.

```
만약 누락된 법인이 있으면, 그 담당자 앞으로 재요청 메일 초안을 작성해줘.
본문은 공손한 톤으로 "매출 보고 회신 지연" 리마인드.
Outlook이나 Gmail 초안으로 저장은 하지 말고 텍스트로만 보여줘.
```

## 수강생에게 강조할 포인트

### 1. **"받은편지함을 Claude가 직접 봅니다"** (MCP)
- API 키 설정·스크립트 실행 없이 **자연어 한 줄**로 메일 조회
- Claude가 메일 제목·본문·발신자 정보까지 직접 읽음
- (오늘 커넥터는 첨부 바이너리는 안 내려받지만, 본문에 핵심 데이터가 있으면 그걸로 충분)

### 2. **40법인 확장성**
- 오늘 시연은 8법인. 육아름 차장님 업무는 40법인.
- **로직은 동일** — 메일 수만 늘어나면 됨

### 3. **누락 자동 감지 (선택)**
- 기대 8법인 중 회신 없는 담당자는 별도 시트 "누락"으로 분리 (오늘 시연은 전원 회신 → 비어있음)
- 실제 육아름님 업무처럼 누락이 있을 때 재요청 초안까지 자동 작성 가능 (F2로 연결)

## MCP vs IMAP — 어느 방식을 쓸까

| 기준 | MCP Gmail (claude_ai) | IMAP (폴백) |
|---|---|---|
| **설정 난이도** | 초기 1회 OAuth | 앱 비밀번호 1회 |
| **자연어 대화형** | ✅ "받은편지함 열어줘" | ❌ 스크립트 실행 필요 |
| **제목·본문 조회** | ✅ search_threads / get_thread | ✅ IMAP 표준 |
| **첨부 바이너리 다운로드** | ❌ 미지원 (본문 파싱으로 우회) | ✅ 표준 FETCH |
| **주기적 무인 실행** | ❌ 대화형 | ✅ Windows 스케줄러 |
| **재현성** | Claude 표현에 의존 | 100% 재현 |
| **업무 시나리오** | 강사 질의응답·육아름 임시 조회 | 김인혜 매일 자동 돌기 |

**데모 F1은 MCP로 체험 → 데모 F2는 IMAP/SMTP로 진짜 자동화**가 오늘의 구성.

> 💡 **수강생용 설명**: "오늘 MCP로 본문만 읽고 첨부는 놔뒀어요. 첨부까지 받아야 하면
> IMAP 방식을 쓰는데, 그건 F2 쇼케이스에서 같이 보여드려요. 두 방식이 **서로를 보완**해요."

## 폴백 실행 (MCP 설정 실패 or 시간 부족)

```bash
py week07-hands-on/demos/email_receive/fallback_imap.py
```

동작:
1. `.env`의 Gmail 인증정보로 IMAP 연결
2. "매출 보고" 제목 최근 메일 검색
3. 첨부 엑셀 다운로드 → 통합 → `output/매출취합_2026-03.xlsx` 생성
4. 누락 법인 확인

수강생에게 "이건 스크립트 실행이지만, Claude가 같은 작업을 대화로 했습니다" 설명.
