# 데모 F1 — 법인 회신 메일 자동 취합 (MCP Gmail)

> **수강생이 느낄 것**: "받은편지함 여는 것부터 첨부 열기·복사·붙여넣기까지 자동이네"
> **핵심 메시지**: Claude는 내 Gmail에 **직접** 들어갈 수 있다 (MCP).
> 받은편지함 읽기·분류·첨부 취합이 **한 번의 대화**로 끝난다.
> **시연 시간**: 12~14분

---

## 업무 맥락 (육아름 차장님 직결)

매달 40개 해외법인 담당자가 월말 매출 보고를 **이메일로** 회신.
지금은 40통을 **한 통씩 열어** 첨부 다운로드 → 폴더 정리 → 엑셀 취합 → DB 반영.
이 업무 반나절을 **한 프롬프트**로 끝내는 쇼케이스.

> **오늘 시연은 8법인(강사 Gmail 자체 발송 메일)으로 축소**.
> 40법인 확장도 동일 구조 — "이름만 늘어날 뿐 로직은 같음" 강조.

---

## 시연 흐름

### 사전 준비 (강사)

#### 1. 샘플 메일 발송 (수업 1~2시간 전)
```bash
py week07-hands-on/prep/send_sample_inbox_emails.py
```
→ 강사 Gmail 받은편지함에 8통의 "법인 담당자 매출 보고" 도착

#### 2. MCP Gmail 서버 설정 (1회, 수업 이전)

Claude Code 설정 → MCP Servers → Gmail 서버 추가 (OAuth).

> 참고: MCP Gmail 설치 경로는 클라우드 서비스에 따라 다름.
> Claude Code 공식 문서 또는 `claude mcp add ...` 명령 참고.

설정 어려우면 **IMAP 폴백**으로 진행 (아래 "폴백 실행" 참고).

---

### 시연 프롬프트 1 — 조회

```
내 Gmail 받은편지함에서 제목이 "매출 보고"를 포함하는 최근 메일을 찾아서
보낸 날짜·제목·첨부파일명을 목록으로 보여줘.
```

**기대 결과**: 8통이 표로 나열됨 (CN01, DE01, GB01, IN01, JP01, TH01, US01, VN01 순)

**수강생 반응 지점**: "저 메일들 전부 Claude가 직접 본 거네?"

### 시연 프롬프트 2 — 첨부 취합

```
위 메일들의 첨부 엑셀을 모두 다운로드해서 한 파일에 통합해줘.

요구사항:
- 각 엑셀에서 2026-03 매출 데이터 추출 (법인코드·통화·금액)
- 8법인의 매출을 원화(KRW)로 환산 (환율은 인터넷에서 최신 기준, 또는 Claude 추정치)
- week07-hands-on/demos/email_receive/output/매출취합_2026-03.xlsx로 저장
  - 시트1: 전체 (법인명·통화·현지금액·KRW환산·담당자)
  - 합계 행 (KRW)
  - 헤더 짙은 남색 배경

- 만약 법인이 누락됐다면(기대 8법인 중 회신 없음) 시트2에 "누락 리스트" 기록
```

### 시연 프롬프트 3 — 누락 확인·후속 조치

```
만약 누락된 법인이 있으면, 그 담당자 앞으로 재요청 메일 초안을 작성해줘.
본문은 공손한 톤으로 "매출 보고 회신 지연" 리마인드.
Outlook이나 Gmail 초안으로 저장은 하지 말고 텍스트로만 보여줘.
```

## 수강생에게 강조할 포인트

### 1. **"받은편지함을 Claude가 직접 봅니다"** (MCP)
- API 키 설정·스크립트 실행 없이 **자연어 한 줄**로 메일 조회
- Claude가 메일 내용·첨부까지 접근

### 2. **40법인 확장성**
- 오늘 시연은 8법인. 육아름 차장님 업무는 40법인.
- **로직은 동일** — 메일 수만 늘어나면 됨

### 3. **누락 자동 감지**
- 8법인 중 회신 안 한 담당자 자동 식별
- 재요청 초안까지 자동 작성 (다음 데모 F2로 자연스럽게 연결)

## MCP vs IMAP — 어느 방식을 쓸까

| 기준 | MCP Gmail | IMAP (폴백) |
|---|---|---|
| **설정 난이도** | 초기 1회 OAuth | 앱 비밀번호 1회 |
| **자연어 대화형** | ✅ "받은편지함 열어줘" | ❌ 스크립트 실행 필요 |
| **주기적 무인 실행** | ❌ 대화형 | ✅ Windows 스케줄러 |
| **재현성** | Claude 표현에 의존 | 100% 재현 |
| **업무 시나리오** | 강사 질의응답·육아름 임시 조회 | 김인혜 매일 자동 돌기 |

**데모 F1은 MCP로 체험 → 데모 F2는 IMAP/SMTP로 진짜 자동화**가 오늘의 구성.

## 폴백 실행 (MCP 설정 실패 or 시간 부족)

```bash
py week07-hands-on/demos/email_receive/fallback_imap.py
```

동작:
1. `.env`의 Gmail 인증정보로 IMAP 연결
2. "매출 보고" 제목 최근 메일 검색
3. 첨부 엑셀 다운로드 → 통합 → `output/매출취합_2026-03.xlsx` 생성
4. 누락 법인 확인

수강생에게 "이건 스크립트 실행이지만, Claude가 같은 작업을 대화로 했습니다" 설명.
