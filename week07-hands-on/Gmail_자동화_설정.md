# Gmail 자동화 설정 가이드 — 앱 비밀번호 발급부터 첫 테스트까지

> **수강생 여러분을 위한 문서입니다.** 오늘 쇼케이스 F1·F2에서 본 "Claude가 받은편지함을 직접 보고 / 자동으로 메일을 보내는" 기능을 **본인 업무에 적용**하려면 이 과정을 1회 거쳐야 합니다.
> 15분이면 끝납니다.

---

## 왜 이 설정이 필요한가

### 일반 Gmail 비밀번호는 자동화에 못 씁니다

2022년 이후 Google은 **일반 비밀번호로 SMTP/IMAP 로그인하는 것을 보안상 차단**했습니다.
"비밀번호 맞는데 왜 안 되지?" → 당연합니다. 막아놨으니까.

### 앱 비밀번호란

- 자동화 전용으로 따로 발급하는 **16자리 영문 코드**
- 일반 비밀번호와 **별개**로 존재 (일반 비밀번호 바뀌어도 앱 비밀번호는 그대로)
- 앱 단위로 따로 만들 수 있음 (Claude Code용, 업무 스크립트용 등)
- 언제든 **개별 폐기 가능** (유출되어도 일반 비밀번호는 안전)

---

## 3단계 설정 (15분)

### Step 1 — 2단계 인증 활성화 (5분)

앱 비밀번호는 **2단계 인증이 켜져 있어야만** 발급 가능합니다.

1. https://myaccount.google.com/security 접속
2. "Google에 로그인" 섹션에서 **2단계 인증** 찾기
3. 꺼져 있으면 → **시작하기** → 본인 휴대폰 번호 입력 → SMS 인증코드로 확인
4. "2단계 인증: **사용 설정됨**" 메시지 확인하면 완료

> 이미 켜져 있는 분은 Step 2로 바로.

---

### Step 2 — 앱 비밀번호 발급 (3분)

1. https://myaccount.google.com/apppasswords 접속
   - 안 열리면 Step 1의 2단계 인증이 제대로 안 된 것. 확인 후 재시도.
2. "앱 이름" 입력란에 식별 가능한 이름 입력 (예: `내업무자동화`, `Claude Code`)
3. **만들기** 또는 **생성** 클릭
4. **노란 박스에 16자리 영문 코드**가 표시됨
   예: `dkqr wxnv mlpz hyub`
5. **이 코드를 복사해두세요.** 창 닫으면 다시 볼 수 없습니다 (분실 시 재발급만 가능).

> 공백(`dkqr wxnv...`)은 무시해도 되고 붙여도 됩니다. `dkqrwxnvmlpzhyub`로 써도 OK.

---

### Step 3 — `.env` 파일에 저장 (2분)

#### 본인 업무 폴더에 `.env` 만들기

```
C:\바탕화면\내업무자동화\.env
```

#### 파일 내용 (복붙해서 쓰세요)

```
# Gmail 자동화용 인증정보 — 이 파일은 절대 공유하지 마세요
GMAIL_USER=본인이메일@gmail.com
GMAIL_APP_PASSWORD=여기에16자리앱비밀번호
```

실제 예시:
```
GMAIL_USER=hong.gildong@gmail.com
GMAIL_APP_PASSWORD=dkqr wxnv mlpz hyub
```

#### ⚠️ 보안 주의사항

- `.env`는 **절대 공유하지 마세요** (Slack·이메일·GitHub 모두)
- 업무 폴더를 Git으로 관리할 경우 `.gitignore`에 `.env` 추가
- 앱 비밀번호가 의심스러운 상황에 노출되면 즉시 Google Account에서 **폐기** 후 재발급

---

### Step 4 — 첫 테스트 (5분)

Claude에게 이렇게 말하세요:

```
내 업무 폴더에 .env 파일을 만들었어 (GMAIL_USER, GMAIL_APP_PASSWORD 있음).
이 계정으로 Gmail SMTP 로그인이 정상 동작하는지 테스트하는 작은 Python 코드를 만들어서
실행해줘. 발송은 하지 말고 로그인 성공/실패만 확인.
```

Claude가 약 10줄짜리 스크립트를 만들어 실행합니다. 결과:
- **성공**: `SMTP login OK` 같은 메시지
- **실패**: `535 5.7.8 Bad Credentials` → 아래 "흔한 실패" 참고


### 업무 시나리오 2 — F1 응용 (육아름 차장님 스타일)

**본문만 있으면 되는 경우 (간편, MCP)**:
```
내 Gmail에서 제목에 "월말 보고" 포함된 최근 7일치 메일 찾아서
본문에 적힌 숫자들을 표로 정리해줘.
(Claude.ai Gmail 커넥터 사용)
```

**첨부 파일까지 받아야 하는 경우 (IMAP)**:
```
내 받은편지함에서 제목에 "월말 보고"가 들어간 메일을 최근 7일치 찾아서
첨부 엑셀 파일을 자동으로 다운로드하고 한 파일로 취합해줘.
IMAP 방식으로. (.env에 자격정보 있음)
```

> 💡 MCP는 자연어 한 줄이면 되지만 **첨부 다운로드는 불가**.
> 첨부까지 꼭 필요하면 IMAP 경로. 업무 성격에 맞춰 고르세요.

## 흔한 실패 & 해결

| 증상 | 원인 | 해결 |
|---|---|---|
| `535 5.7.8 Bad Credentials` | 앱 비밀번호 오타 | Step 2 다시 발급 → 복사 → 재입력 |
| `535 5.7.14 Please log in via your web browser` | 새 IP에서 로그인 의심 | Gmail 받은편지함의 **보안 경고** 메일 열어 "저였습니다" 확인 |
| `apppasswords 페이지 안 열림` | 2FA 미활성화 | Step 1 완료 후 재시도 |
| `Connection refused` | 네트워크/방화벽 | 회사망에서 SMTP 포트(465/587) 막힌 경우 IT팀 확인 |
| 메일은 갔는데 안 옴 | 스팸 처리 | 받는 쪽 **스팸함** 확인 (본인 → 본인 발송은 종종 스팸 분류) |

## 학교메일(Google Workspace) 쓰시는 분

학교/회사 Gmail은 **관리자 정책에 따라 앱 비밀번호가 차단**될 수 있습니다.
그 경우:
1. apppasswords 페이지가 아예 안 열리거나 "사용 불가" 표시
2. IT/학교 관리자에게 "앱 비밀번호 허용" 또는 "개인 Gmail 계정 사용 가능" 문의
3. 대안: **OAuth 방식** (더 복잡하지만 관리자가 막아도 회피 가능) — 필요 시 Claude에게 "OAuth로 Gmail 연결해줘" 지시

본인 **개인 Gmail**이 있으면 그걸로 하는 게 가장 간단합니다. 업무 결과물만 학교 메일로 전달하면 됨.
