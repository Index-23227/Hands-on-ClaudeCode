# 검증 체크리스트 — Claude 결과물 받으면 이것부터

> Claude가 만든 결과물은 **그럴듯해 보여도 틀릴 수 있습니다.**
> 아래 5가지만 돌려도 대부분의 오류는 잡힙니다.
> 실제 업무에 쓰기 전엔 **반드시** 통과시키세요.

---

## ① 행 개수 맞나?

원본 데이터의 행 수와 결과물의 행 수가 논리적으로 맞는지 확인.

- 원본 100건 → 결과물도 100건이어야 하는 작업인가, 필터된 결과인가?
- 필터된 거라면 "왜 X건이 나왔는지" 설명이 가능해야 함

**이상하면**:
```
결과가 ○건인데 원본은 ○건이었어. 왜 달라졌어? 누락된 행이 있어?
```

---

## ② 합계 맞나?

숫자 컬럼의 합계를 **원본과 결과물 양쪽에서** 내보고 비교.

- 엑셀이면 합계 행에 `=SUM()` 걸어서 직접 확인
- 통화 단위 섞였으면 단위별로 따로 합산

**이상하면**:
```
합계가 ○원인데 원본은 ○원이었어. 어디서 차이가 났는지 찾아줘.
```

---

## ③ 샘플 1~2건 수기 대조

무작위로 1~2건 뽑아서 **손으로 계산·확인**해서 같은지 보기.

- 변동률 계산이라면 전자계산기로 직접 계산
- 환율 환산이라면 당일 환율 곱해보기
- "이건 확실히 맞다"는 샘플이 있어야 나머지도 믿을 수 있음

**이상하면**:
```
○○ 법인의 수치가 ○인데 내가 계산해보니 ○이 나와. 어떻게 계산한 거야?
```

---

## ④ 포맷(서식) 확인

숫자가 **숫자로 인식**되고 있나? 문자열로 저장되면 나중에 합계 안 됨.

- 금액 컬럼이 오른쪽 정렬인지 (숫자는 오른쪽, 문자는 왼쪽)
- 천단위 콤마·소수점·퍼센트 의도대로인지
- 날짜가 텍스트 "2026-04-01"인지 진짜 날짜인지

**이상하면**:
```
금액 컬럼이 문자열로 들어갔어. 숫자로 바꾸고 천단위 콤마 서식 넣어줘.
```

---

## ⑤ 예외 케이스 확인

평소 같으면 무시하는 케이스들이 결과물에 제대로 반영됐는지.

- 빈 칸(NULL)과 0을 구분하는가? (매출 0원과 "아직 보고 안 함"은 다름)
- 주말·공휴일 데이터가 빠져있거나 이상하게 들어가 있지 않은가?
- 통화가 다른 법인을 그냥 더해버리진 않았나?
- 음수(마이너스)가 제대로 처리됐나?

**이상하면**:
```
○○ 법인은 데이터가 없는데 0으로 처리됐어. 빈 칸은 0이 아니라 "N/A"로 표시해줘.
```

## 최종 통과 기준

위 5개가 **전부 통과**해야 실제 업무에 사용합니다.

| 항목 | 통과? |
|---|---|
| ① 행 개수 | ☐ |
| ② 합계 | ☐ |
| ③ 샘플 수기 대조 | ☐ |
| ④ 포맷 | ☐ |
| ⑤ 예외 케이스 | ☐ |

**하나라도 실패**하면 Claude에게 그 항목을 짚어서 "이 부분이 틀렸어, 고쳐줘"라고 말하세요.
모든 수정 후엔 **체크리스트를 처음부터 다시** 돌립니다.
