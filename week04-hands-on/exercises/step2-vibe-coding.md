# Step 2: 바이브코딩 체험 (25분)

> 방금 만든 CLAUDE.md를 활용해서, AI에게 코드를 시켜봅니다.
> "바이브코딩" = 한국어로 설명하면 AI가 코드를 만들어주는 것

---

## 바이브코딩이란?

```
내가 할 일:  한국어로 "이거 해줘" 설명
AI가 할 일:  Python 코드를 만들어줌
내가 할 일:  실행 버튼만 누르기
```

코드를 외울 필요 없습니다. **한국어로 잘 설명하는 것**이 핵심입니다.

---

## 실습 A: 체험 ③ — 하네스 완성 후 같은 프롬프트 (10분)

Step 0에서 같은 프롬프트를 2번 시켜봤죠?
- 체험 ① claude.ai → 가짜 데이터, 코드만 보여줌
- 체험 ② Claude Code (MD 없이) → 파일은 읽지만, 우리 요구사항을 모름

이제 CLAUDE.md가 있는 상태에서 **세 번째**로 시켜봅니다.

### Claude Code에서 똑같이 입력

```
경비 정리하는 코드 만들어줘.
```

**체험 ②와 뭐가 달라졌는지 관찰하세요:**

- AI가 시작할 때 **CLAUDE.md를 자동으로 읽음** (체험 ②에서는 이게 없었음)
- 파일명이 **`[재경]경비정리_202603_v1.xlsx`** (우리 팀 양식대로)
- 첫 번째 시트가 **"경비총괄표"** (팀장 보고용 정해진 양식)
- 부서 순서가 **영업1팀→영업2팀→관리팀→합계** (가나다순이 아님!)
- 접대비 120,000원에 **"※ 증빙 첨부 필수"**, 95,000원에 **"※ 팀장 사전승인"** (금액 구간별 비고)
- 개인카드 정산 시트에 **"정산상태=미정산"** 컬럼 (부서장 결재 프로세스)

### 3단계 비교 완성

메모장에서 체험 ①, ② 결과를 꺼내서 비교해보세요:

| | ① claude.ai | ② Claude Code (MD 없이) | ③ 지금 (MD 있게) |
|--|------------|------------------------|------------------|
| 파일 접근 | × 가짜 데이터 | ○ 실제 파일 읽음 | ○ 실제 파일 읽음 |
| 코드 실행 | × 코드만 보여줌 | ○ 실행까지 해줌 | ○ 실행까지 해줌 |
| 파일명 | × | × 아무 이름 | **○ `[재경]경비정리_202603_v1.xlsx`** |
| "경비총괄표" 시트 | × | × 없음 | **○ 정해진 양식대로** |
| 일반/특별경비 분류 | × | × 안 함 | **○ 사내 규칙대로** |
| 접대비 구간별 비고 | × | × 기준 모름 | **○ 10만원↑ / 5~10만원 구분** |
| 정산상태 = "미정산" | × | × 프로세스 모름 | **○ 부서장 결재용** |
| 부서 순서 | × | × 가나다순 | **○ 영업1팀→영업2팀→관리팀→합계** |

> **①→② = 도구의 차이** (파일을 읽고, 코드를 실행할 수 있게 됨)
> **②→③ = 하네스의 차이** (파일명 양식, 시트 구성, 경비 규칙, 정산 프로세스 — 회사 고유 양식이라 데이터만 봐서는 절대 모름)
>
> 하네스의 핵심: **같은 도구, 같은 프롬프트인데 "우리 회사 양식"대로 결과가 나온다.**

## 실습 B: 결과 확인하고 수정 시켜보기 (10분)

생성된 `[재경]경비정리_202603_v1.xlsx`를 열어서 확인해보세요.
혹시 양식이 조금 다르면, AI에게 수정을 시키면 됩니다:

```
경비총괄표 시트에서 합계 행 글씨를 굵게 해줘.
```

```
전체내역 시트에서 특별경비 행의 배경색을 노란색으로 해줘.
```

CLAUDE.md가 있으면 "어떤 시트의 어떤 데이터"를 말하는지 AI가 바로 이해합니다.
하네스 없이는 이런 후속 요청도 매번 처음부터 설명해야 합니다.

## 만약 코드가 에러가 난다면?

걱정하지 마세요. 에러 메시지를 AI에게 보여주면 됩니다.

```
이 코드를 실행했더니 이런 에러가 나:

[에러 메시지 복사 붙여넣기]

고쳐줘.
```

이것도 바이브코딩의 일부입니다. **에러가 나면 → AI한테 고쳐달라고 하기.**
