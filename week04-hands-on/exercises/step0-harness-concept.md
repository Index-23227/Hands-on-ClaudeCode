# Step 0: 하네스(Harness) 개념 이해하기 (20분)

> AI를 "그냥 쓰는 것"과 "체계적으로 제어하는 것"은 완전히 다릅니다.
> 이 차이를 이해하면, 왜 CLAUDE.md를 만들어야 하는지 납득이 됩니다.

---

## 하네스가 뭔가요?

**하네스(Harness) = AI를 내 업무에 맞게 제어하는 장치**

공사 현장에서 안전 하네스를 생각해보세요.
하네스 없이 일하면? 위험하고, 방향도 제멋대로입니다.
하네스를 착용하면? 안전하고, 원하는 범위 안에서 일할 수 있습니다.

AI도 마찬가지입니다:

| | 하네스 없이 (그냥 채팅) | 하네스 있게 (제어된 환경) |
|--|----------------------|------------------------|
| AI가 아는 것 | 아무것도 모름 — 매번 처음부터 설명 | 내 업무, 파일, 규칙을 미리 알고 있음 |
| 결과 품질 | 일반적인 답변 — 우리 상황과 안 맞음 | 우리 상황에 맞는 정확한 결과 |
| 반복 작업 | 매번 같은 설명을 다시 해야 함 | 한 번 설정하면 계속 적용 |
| 실수 방지 | AI가 알아서 판단 — 위험할 수 있음 | 규칙을 정해서 실수를 막음 |

## 비유로 이해하기

### 비유 1: 파견 직원

```
하네스 없이 = 파견 직원에게 "일 해주세요" 라고만 말하기
                → 뭘 해야 하는지 모르니까 엉뚱한 일을 함

하네스 있게 = 파견 직원에게 업무 매뉴얼 + 규칙집을 주기
                → 우리 방식대로 정확하게 일함
```

### 비유 2: 내비게이션

```
하네스 없이 = "서울 가주세요" (어디로 가는 서울? 어떤 길로?)
하네스 있게 = "서울 강남구 테헤란로 152, 고속도로 이용, 톨게이트 회피"
```

## Claude Code의 하네스 구성요소

Claude Code에서 "하네스"는 이런 것들로 이루어집니다:

```
┌─────────────────────────────────────────┐
│            Claude Code 하네스            │
│                                         │
│  ① CLAUDE.md     = 업무 설명서           │
│     "나는 재경팀이고, 매월 경비를 정리해"    │
│                                         │
│  ② 프로젝트 파일  = 실제 데이터            │
│     data/ 폴더의 엑셀 파일들을 직접 봄      │
│                                         │
│  ③ 설정(Settings) = 행동 규칙             │
│     "코드는 항상 한국어 주석으로"           │
│                                         │
│  ④ 훅(Hooks)     = 자동 점검              │
│     "코드를 만들면 자동으로 검사 실행"       │
│                                         │
└─────────────────────────────────────────┘
```

**오늘 실습에서는 ①과 ②를 직접 만들어봅니다.**
(③④는 나중에 익숙해지면 추가하면 됩니다)

## 직접 체험: 3단계로 차이 느껴보기

같은 프롬프트를 3가지 환경에서 시켜봅니다.
프롬프트는 전부 동일: **"경비 정리하는 코드 만들어줘."**

```
체험 ①  claude.ai (도구 없음, 하네스 없음)        ← 지금
체험 ②  Claude Code, CLAUDE.md 없이 (도구만 있음)  ← 지금
체험 ③  Claude Code, CLAUDE.md 있게 (도구 + 하네스) ← Step 2에서
```

---

### 체험 ①: claude.ai에서 시키기

[claude.ai](https://claude.ai) 에 접속해서 입력:

```
경비 정리하는 코드 만들어줘.
```

**관찰 포인트:**
- 우리 파일(경비내역.xlsx)을 모르니까 가짜 예시 데이터를 만들어냄
- 코드만 보여주고 끝 — 직접 복사해서 실행해야 함

> 결과를 **메모장에 "체험①"로 복사**해두세요.

---

### 체험 ②: Claude Code에서 CLAUDE.md 없이 시키기

VSCode 터미널에서 Claude Code를 열고, **똑같이** 입력:

```
경비 정리하는 코드 만들어줘.
```

**체험 ①과 달라진 점을 관찰하세요:**
- data/ 폴더에 경비내역.xlsx가 있는 걸 발견하고 직접 열어봄
- 실제 컬럼명(일자, 부서, 항목, 금액, 결제방법)에 맞춰서 코드를 만듦
- 코드를 실행까지 해서 결과가 바로 나옴

**하지만 아직 부족한 점도 관찰하세요:**
- 파일명이 아무 이름임 (우리 팀은 `[재경]경비정리_202603_v1.xlsx` 양식을 씀)
- "경비총괄표" 시트가 없음 (팀장 보고용 정해진 양식이 있는데 모름)
- "일반경비/특별경비" 분류를 안 함 (사내 규칙을 모르니까)
- 접대비 10만원 이상에 "증빙 첨부 필수" 표시를 안 함 (기준 금액을 모르니까)
- 부서 순서가 가나다순임 (우리 보고서는 영업1팀→영업2팀→관리팀→합계 고정)

> 결과를 **메모장에 "체험②"로 복사**해두세요.

---

### 체험 ③은 Step 2에서!

Step 1에서 CLAUDE.md를 만든 뒤, Step 2에서 **또 똑같은 프롬프트**를 시켜봅니다.
CLAUDE.md에 파일명 양식, 시트 구성, 사내 경비 규칙, 비고 표시 기준을 적어놓으면,
**같은 한 줄 프롬프트인데 "우리 회사 양식"대로 결과가 나옵니다.**

## 3단계 비교 미리보기

| | ① claude.ai | ② Claude Code (MD 없이) | ③ Claude Code (MD 있게) |
|--|------------|------------------------|------------------------|
| 프롬프트 | 경비 정리하는 코드 만들어줘 | **똑같음** | **똑같음** |
| 파일 접근 | × 가짜 데이터 | ○ 직접 열어봄 | ○ 직접 열어봄 |
| 코드 실행 | × 코드만 보여줌 | ○ 실행까지 해줌 | ○ 실행까지 해줌 |
| 파일명 | × | × 아무 이름 | ○ `[재경]경비정리_202603_v1.xlsx` |
| "경비총괄표" 시트 | × | × 없음 | ○ 정해진 양식대로 |
| 일반경비/특별경비 분류 | × | × 안 함 | ○ 사내 규칙대로 |
| 접대비 구간별 비고 | × | × 기준 모름 | ○ 10만원↑ "증빙 첨부 필수" |
| | | | 　5~10만원 "팀장 사전승인" |
| 정산상태 = "미정산" | × | × 프로세스 모름 | ○ 부서장 결재용 |
| 부서 순서 | × | × 가나다순 | ○ 영업1팀→영업2팀→관리팀→합계 |

> **①→② = 도구의 차이** (파일을 읽고, 코드를 실행할 수 있게 됨)
> **②→③ = 하네스의 차이** (파일명 양식, 시트 구성, 경비 규칙, 정산 프로세스 — 회사 고유 양식이라 데이터만 봐서는 절대 모름)

## 핵심 정리

```
AI를 잘 쓴다 = 좋은 하네스를 만든다
           = AI가 일할 환경을 잘 세팅한다

1. CLAUDE.md로 업무를 설명하고        ← Step 1에서 함
2. 실제 데이터를 같은 폴더에 두고      ← 이미 data/에 있음
3. AI에게 일을 시킨다                 ← Step 2~3에서 함
```

**"프롬프트를 잘 쓰는 것"보다 "하네스를 잘 만드는 것"이 더 중요합니다.**
프롬프트는 매번 바뀌지만, 하네스는 한 번 만들면 계속 쓸 수 있으니까요.
